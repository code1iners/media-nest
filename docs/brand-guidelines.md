# MyTube Extract 브랜드 가이드라인

## 브랜드 배경과 목표

MyTube Extract는 개인 사용자가 가진 YouTube URL 또는 로컬 영상에서 필요한 파일을 준비하는 보조 도구다.

브랜드 목표는 복잡한 설정과 API 경로를 드러내지 않고, 사용자가 필요한 결과물을 빠르게 요청·확인·다운로드하게 만드는 것이다.

핵심 약속은 다음과 같다.

> 복잡한 다운로드 없이, 한 번에 추출.

이 약속은 Chrome 확장과 web 앱에 동일하게 적용한다. API는 사용자에게 노출되는 안내·오류 문구에서 이 약속을 훼손하지 않는다.

## 핵심 사용자

첫 번째 사용자는 개인 사용자다.

- YouTube URL에서 오디오 또는 비디오 파일을 준비하려는 사람
- 로컬 영상에서 영어 SRT를 만들려는 사람
- 세부 기술 설정보다 빠른 완료와 명확한 상태를 원하는 사람

개발자·파워 유저는 현재 제품을 사용할 수 있지만, 브랜드의 기본 언어와 화면 구조는 이들을 우선하지 않는다.

## 포지셔닝과 경쟁 대안

MyTube Extract는 링크를 붙여넣어 결과 파일을 준비하는 개인용 추출 도구다.

- `cobalt`처럼 단일 행동을 빠르게 시작하는 흐름을 참고한다.
- `4K Video Downloader Plus`처럼 기능을 나열하거나 전문 도구·대량 관리 도구처럼 보이려 하지 않는다.
- URL 입력, 형식 선택, 결과 상태, 다운로드라는 현재 제품 범위를 명확히 보여 준다.

지원 범위 밖의 기능, 자동화, 계정 연동, 무제한·보장 같은 표현은 사용하지 않는다.

## 브랜드 약속과 성격

브랜드 성격은 다음 세 가지다.

- 빠른: 사용자는 한 화면에서 다음 행동을 바로 알 수 있다.
- 장난기 있는: 픽셀 콘솔과 제한된 밈 카피로 발견의 재미를 만든다.
- 명확한: 입력, 정책, 진행 상태, 실패 이유는 도구답게 직설적으로 전달한다.

가벼운 표현이 기능의 신뢰도나 저작권·플랫폼 정책 안내를 흐리지 않아야 한다.

## 보이스와 카피 원칙

기본 언어는 한국어다. 기능명, URL, 파일 형식, 오류 코드 등 기술 용어는 필요한 경우 원문을 유지한다.

| 상황 | 원칙 | 예시 |
| --- | --- | --- |
| 배너·완료 | 짧은 밈 톤 허용 | `이 영상은 이제 제 겁니다` |
| CTA | 동작을 그대로 쓴다 | `추출 시작`, `현재 탭 사용`, `영어 SRT 생성` |
| 입력·설정 | 기능을 정확히 설명한다 | `추출 URL`, `오디오`, `비디오` |
| 처리 상태 | 현재 상태와 다음 결과를 말한다 | `파일을 준비하고 있어요.` |
| 오류·정책 | 유머 없이 원인과 행동을 전달한다 | `지원하는 YouTube URL을 입력하세요.` |

피해야 할 표현:

- YouTube 공식 서비스·제휴·승인을 암시하는 표현
- 저작권 또는 플랫폼 정책을 무시해도 된다는 표현
- 처리 결과를 보장하거나 실제 범위보다 넓게 말하는 표현
- 오류·실패 상황의 밈 카피

## 네이밍과 핵심 메시지

제품명은 `MyTube Extract`다.

- 첫 노출에서는 제품명을 완전한 형태로 쓴다.
- 좁은 UI에서는 `Extract`를 단독 제품명처럼 사용하지 않는다.
- `YouTube`는 지원 URL의 출처를 설명할 때만 쓴다.
- 제품명·아이콘·카피를 YouTube 공식 로고, 재생 버튼, 고유 색 조합과 결합하지 않는다.

핵심 메시지 우선순위:

1. `복잡한 다운로드 없이, 한 번에 추출.`
2. `이 영상은 이제 제 겁니다`
3. 기능별 설명: `YouTube URL을 제출하면 순서대로 파일을 준비합니다.` / `내 디바이스의 영상에서 영어 SRT를 생성합니다.`

## 비주얼 시스템

MyTube Extract는 픽셀 콘솔 UI를 공통 시각 언어로 사용한다.

- 짙은 navy 기반의 캔버스와 패널
- deep violet은 주요 행동·선택
- teal은 활성·진행·포커스
- amber는 완료·보상 강조
- red는 오류·주의
- 회색 계열은 구조·보조 정보
- 작은 격자, 각진 테두리, 단단한 하단 그림자로 픽셀 콘솔의 촉감을 만든다

색은 장식이 아니라 상태를 구분하는 역할로 사용한다. violet·teal·amber·red를 같은 요소에 동시에 경쟁시키지 않는다.

공통 color와 typography Foundation은 [MyTube Extract Design System](DESIGN.md)을 기준으로 한다. 기존 구현의 token은 문서의 목표 상태를 구현하기 전까지 유지한다.

## 로고와 아이콘

현재 공식 식별 자산은 픽셀 추출기 아이콘이다.

- 추출 팔, 집게, 캡슐의 단순한 픽셀 실루엣을 유지한다.
- 작은 크기에서도 “미디어에서 무언가를 꺼낸다”는 의미가 읽혀야 한다.
- 앱 아이콘은 배경·패널·mint 팔·gold/red 캡슐의 현재 구조를 유지한다.
- 새로운 로고 워드마크, 최소 크기, 안전 여백은 전용 원본 자산이 생긴 뒤 정의한다.

YouTube 로고, 공식 재생 버튼 형태, 공식 red/white 조합을 아이콘에 직접 사용하지 않는다.

## 타이포그래피

- web과 Chrome 확장: `LanaPixel`을 UI 폰트로 사용하고, 시스템 sans-serif를 fallback으로 둔다.
- 숫자, URL, 파일명, 오류 메시지도 `LanaPixel`을 사용하되, 대비·크기·행간으로 읽기 쉬워야 한다.
- font weight를 계층 수단으로 삼지 않고 size·색·공간을 사용한다.

## 적용 예시

### Chrome 확장

- 상단: 제품명, 보조 설명, 밈 카피, 픽셀 추출기
- 본문: `추출 URL` → 형식 선택 → 선택 옵션 → `추출 시작`
- 정책 문구: 실행 전 노출
- 오류: 입력값 또는 서버 상태를 먼저 설명하고 해결 행동을 제시

### Web

- 공통 hero: 제품명과 route별 기능 설명
- `영상 추출`: URL 제출과 파일 준비 상태를 중심으로 구성
- `자막 추출`: 로컬 파일 선택과 영어 SRT 생성 상태를 중심으로 구성
- 하단 탭: 기능 이동만 담당하고, 새로운 브랜드 메시지를 중복하지 않음

### API 안내

- API 문서와 오류 메시지는 브랜드 카피보다 계약 정확성을 우선한다.
- 사용자 노출 오류는 짧은 한국어 안내와 필요한 기술 상세를 분리한다.

## 접근성

- 본문·입력·정책·상태 텍스트는 색만으로 의미를 전달하지 않는다.
- 상태는 라벨, 아이콘 또는 문장으로 함께 구분한다.
- 키보드 포커스는 mint outline처럼 명확하게 보인다.
- 주요 CTA·입력·오류 텍스트의 대비는 구현 변경 시 WCAG AA 기준으로 검증한다.
- hover만으로 상태를 전달하지 않는다.

## 금지 사례

- 픽셀 콘셉트 때문에 URL, 파일명, 오류 메시지의 가독성을 희생하는 것
- 모든 문구에 밈 톤을 적용하는 것
- 지원하지 않는 플랫폼·품질·처리 결과를 약속하는 것
- YouTube와의 공식 관계를 암시하는 것
- 현재 구현과 다른 색상·폰트·아이콘 규격을 문서만으로 확정하는 것
